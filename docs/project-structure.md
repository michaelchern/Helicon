# Agent Context: Project Structure

Purpose: compact operational map for AI agents. Human-facing explanations live
in `docs/zh/project-structure.md`; treat that Chinese file as the source of
truth when syncing meaning.

## Source Contract

- Human source: `docs/zh/project-structure.md`
- Agent target: `docs/project-structure.md`
- Do not mirror the Chinese prose sentence by sentence.
- Normalize Chinese intent into concise search rules, ownership boundaries, and
  maintenance constraints.
- Preserve paths, commands, script names, target names, and public API names.

## Required Read Order

1. `docs/architecture.md`
2. `docs/project-structure.md`

- `.git/`: Git metadata; do not edit by hand.

## Directory Contracts

| Path | Contract |
| --- | --- |
| `docs/zh/` | Human-editable Chinese source docs. Prefer these when user intent and English docs conflict. |
| `docs/` | Derived English agent context. Keep concise, operational, and searchable. |
| `include/` | Public headers and stable API boundary. Check compatibility before changing signatures. |
| `src/` | Production implementation. Keep internal behavior here. |
| `tests/` | Executable validation. Update when behavior changes or bugs are fixed. |
| `scripts/` | Standard automation entry points. Prefer these over ad hoc commands. |
| `examples/` | Usage examples and minimal demos. May contain only `.gitkeep` during scaffold stage. |
| `skills/` | Repository-local Codex workflows. Use for repeated agent tasks. |
| `.github/workflows/` | GitHub Actions CI definitions. Inspect for CI and PR-check issues. |
| `.codex/` | Local Codex configuration and reserved policies. Change only for Codex configuration tasks. |
| `third_party/` | Vendored code or external dependencies. Require explicit justification before adding dependencies. |

| Vulkan backend work | `src/backends/vulkan/` | shared `src/backends/` |

## Standard Commands

```powershell
powershell -ExecutionPolicy Bypass -File scripts/build.ps1
powershell -ExecutionPolicy Bypass -File scripts/test.ps1
```

Use these commands for validation unless the task explicitly requires a
different entry point.

## Agent Behavior Rules

- If the user edits or references Chinese docs, treat them as source intent.
- Do not ask the user to write English docs; translate and normalize them.
- English docs should optimize for retrieval and action, not narrative reading.
- Keep generated outputs out of source control.
- Keep directory responsibilities single-purpose: public API in `include/`,
  implementation in `src/`, tests in `tests/`, scripts in `scripts/`.
